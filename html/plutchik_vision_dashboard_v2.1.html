API layer (FastAPI)

Implemented in plutchik_erc_dashboard/inference_server.py:

  • Live POST /predict — multi-task output + optional assessment (HITL signal).
  • Live POST /predict/batch — stateless batch scoring.
  • Live POST /predict/arc — dialogue arc + KL-based inflection hints.
  • Live POST /explain — Captum IG on the full [CONTEXT]…[CURRENT]… string; returns context_span_top / current_span_top.
  • Live X-API-Key + per-IP rate limit (429) via verify_api_key.

Training — DAT / GRL warmup

train_v2.py + utils/trainer.py: warmup_steps = warmup_epochs × len(train_loader). With epochs=5 and warmup_epochs=1, warmup is one full epoch (≈20% of all optimizer steps).

models/multitask_emotion_model.py — scenario discriminator is 2 logits (binary domain), not raw multi-class scenario IDs.

Analyst dashboard (Streamlit)

  • Comparative trajectories + per-conversation tables + emotion histograms (app.py).
  • CSV batch upload with configurable row cap + distribution / ring / sarcasm profile.
  • Checkbox for full explainability → POST /explain; context vs current IG tables in Model Internals.

CDA / HITL

  • Verified contrastive merge: PLUTCHIK_CDA_JSONL + PLUTCHIK_CDA_MIN_PAIRS (default 200) in train_v2.py.
  • Gate docs: data/processed/ERC/CDA_GATE_README.md · counter scripts/count_cda_verified.py.
  • Human verification CLI: python utils/pair_verifier.py (from dashboard dir).

Extension + compression path
